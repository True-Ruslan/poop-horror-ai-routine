# ASSET_SOURCES.md

## Цель

Этот файл фиксирует источники ассетов, которые можно рассматривать для будущей публикации игры. Перед добавлением любого ассета нужно записать источник, автора, ссылку, лицензию и условия использования в `docs/CREDITS.md`.

Для звуков дополнительно читать `docs/SOUND_DESIGN.md`.

## Предпочтительные лицензии

| Лицензия | Коммерческий релиз | Атрибуция | Комментарий |
|---|---:|---:|---|
| CC0 | Да | Нет | Лучший вариант для прототипа и релиза |
| MIT | Да | Да | Хорошо для кода и инструментов |
| CC BY | Да | Да | Можно при корректном указании автора |

## Не использовать без отдельного решения

| Лицензия | Причина |
|---|---|
| GPL | Может создавать лишние обязательства для релиза |
| AGPL | Слишком рискованно для простого инди-релиза |
| LGPL | Нужна отдельная проверка условий распространения |
| CC BY-NC | Не подходит для коммерческой игры |
| CC BY-ND | Нельзя изменять |
| Sampling+ | Старый и неудобный вариант для релиза |
| Unknown | Нельзя подтвердить права |

## Источники для проверки

- Godot Asset Library — плагины, демо и материалы для Godot. Проверять лицензию каждого пакета.
- Kenney — хороший источник CC0-ассетов.
- OpenGameArt — много ассетов, но у каждого материала своя лицензия.
- Freesound — звуки; брать только CC0 или CC BY.
- Pixabay — можно рассматривать отдельные звуки, но это не CC0; проверять Content License и не распространять звук standalone.
- Itch.io asset packs — использовать только при явно указанной лицензии.

## Звуки

Для первого эпизода нужны короткие, сухие и бытовые звуки:

- уведомление;
- щелчок лампы;
- одиночная клавиша;
- короткий набор на клавиатуре;
- электронный отказ двери;
- пробуждение колонки;
- слабый room tone.

Правило: не добавлять звук, пока не ясно, где он используется и какой event вызывает его.

## Сгенерированные звуки

Сгенерированные звуки можно использовать только если условия сервиса разрешают использование результата в игре.

В `docs/CREDITS.md` записать:

- сервис;
- дату генерации;
- prompt;
- условия использования;
- постобработку;
- место использования.

Если условия сервиса не ясны — звук не добавлять.

## Правила проекта

1. На старте использовать собственные примитивы Godot.
2. Сначала добавлять только CC0-ассеты.
3. Не использовать реальные логотипы компаний и сервисов.
4. Не копировать интерфейсы известных сервисов.
5. Не использовать музыку и звуки из чужих игр, трейлеров или роликов.
6. Если лицензия непонятна — ассет не добавлять.
7. Если звук CC BY — обязательно указать автора.
8. Если звук сгенерирован — обязательно указать сервис и условия.
