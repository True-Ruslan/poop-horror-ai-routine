# GAME_DESIGN.md

## Цель игры

**AI Routine: Last Commit** — короткий атмосферный хоррор от первого лица про разработчика, вечернюю рабочую рутину и постепенное ощущение, что обычные цифровые подсказки стали слишком личными.

## Главная эмоция

```text
Я понимаю, что нужно сделать, но не хочу снова смотреть на монитор.
```

## Основные принципы

1. Игрок всегда получает простую бытовую цель.
2. Страх строится через текст, свет, звук и ожидание.
3. Юмор должен быть сухим и редким.
4. Сначала делаем маленькую рабочую локацию, потом расширяем.
5. Любая новая механика должна проверяться за 1–2 минуты.

## Тон

Игра не должна быть чистой комедией. Ирония нужна только для узнаваемости рабочей рутины: странные формулировки задач, усталые заметки, абсурдные предупреждения сборки. После шутки должно оставаться напряжение.

## MVP

Для первой версии достаточно:

- одна квартира;
- рабочий стол;
- монитор;
- стикер;
- лампа;
- дверь;
- 2–3 scripted events;
- понятный финал.

## Что не добавлять в MVP

- боёвку;
- большого противника;
- сложный инвентарь;
- большие локации;
- длинные диалоги;
- реальные сторонние интеграции.

## Следующий вертикальный срез

1. Сделать монитор многофазным.
2. Добавить мигание лампы после первого сообщения.
3. Добавить интерактивную колонку или телефон.
4. Добавить один лицензированный звук.
5. Обновить `docs/CREDITS.md`.
