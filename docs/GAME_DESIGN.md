# GAME_DESIGN.md

## Цель игры

**AI Routine: Last Commit** — короткий атмосферный хоррор от первого лица про разработчика, вечернюю рабочую рутину и постепенное ощущение, что обычные цифровые подсказки стали слишком личными.

## Главная эмоция

```text
Я понимаю, что нужно сделать, но не хочу снова смотреть на монитор.
```

## Основные принципы

1. Игрок всегда получает простую бытовую цель.
2. Страх строится через текст, свет, звук и ожидание.
3. Юмор должен быть сухим и редким.
4. Сначала делаем маленькую рабочую локацию, потом расширяем.
5. Любая новая механика должна проверяться за 1–2 минуты.
6. Деталь важнее масштаба: один точный щелчок, странная строка или неверная подсказка сильнее большой декорации.
7. Сначала проектируем beat, звук и реакцию объекта, потом пишем код.

## Тон

Игра не должна быть чистой комедией. Ирония нужна только для узнаваемости рабочей рутины: странные формулировки задач, усталые заметки, абсурдные предупреждения сборки. После шутки должно оставаться напряжение.

## Ключевые дизайн-документы

Перед реализацией новых сцен и механик читать:

- `docs/NARRATIVE_DESIGN.md` — вайб, темп, микродетали, классы хоррор-моментов.
- `docs/INTERACTION_MECHANICS.md` — каталог механик взаимодействия.
- `docs/SOUND_DESIGN.md` — звуковой подход, источники, лицензии, backlog звуков.
- `docs/SCENE_BEATS.md` — beat map первого эпизода.

## MVP

Для первой версии достаточно:

- одна квартира;
- рабочий стол;
- монитор;
- стикер;
- лампа;
- дверь;
- 2–3 scripted events;
- 1–2 коротких звука;
- понятный финал.

## Что не добавлять в MVP

- боёвку;
- большого противника;
- сложный инвентарь;
- большие локации;
- длинные диалоги;
- реальные сторонние интеграции.

## Следующий вертикальный срез

1. Сделать монитор многофазным.
2. Добавить мигание лампы после первого сообщения.
3. Добавить один звук уведомления и один щелчок лампы.
4. Добавить интерактивную колонку или телефон.
5. Добавить HUD corruption или изменение подсказки взаимодействия.
6. Обновить `docs/CREDITS.md` при добавлении звука.
