# SCENE_BEATS.md

## Назначение

Документ раскладывает первый эпизод на короткие игровые сцены. Каждая сцена должна быть достаточно маленькой, чтобы её можно было реализовать отдельным PR.

Каждый beat описывает:

- цель игрока;
- объект взаимодействия;
- нормальное объяснение;
- маленькое нарушение;
- ключевой звук;
- следующий шаг.

## Beat 0 — Late night setup

Цель игрока:

```text
Осмотреть рабочее место.
```

Игрок появляется в квартире ночью рядом с рабочим столом.

Детали:

- монитор светит в комнату;
- на столе есть стикер;
- лампа рядом с монитором;
- входная дверь видна, но пока не главная цель;
- HUD показывает первую рабочую задачу.

Звук:

- тихий room tone;
- слабый гул монитора или ПК.

## Beat 1 — Task sticker

Цель игрока:

```text
Прочитать стикер на столе.
```

Нормальное объяснение:

Игрок вспоминает список задач.

Нарушение:

Последняя строка выглядит так, будто её написал не герой.

Пример:

```text
Не доверять ответу, если он слишком уверенный.
```

Следующий шаг:

```text
Открыть рабочий терминал.
```

## Beat 2 — First terminal reply

Цель игрока:

```text
Открыть рабочий терминал.
```

Нормальное объяснение:

Обычный лог сборки и ответ код-агента.

Нарушение:

Предупреждение звучит слишком лично.

Пример:

```text
Build #1047: success
Tests: 0 failed
Warnings: 1

warning: user accepted generated code without reading.
```

Звук:

- обычное уведомление;
- короткий keyboard tick.

Следующий шаг:

```text
Проверить, почему лампа включилась сама.
```

## Beat 3 — Lamp answers terminal

Цель игрока:

```text
Проверить лампу.
```

Нормальное объяснение:

Глюк умной лампы или электрики.

Нарушение:

Лампа реагирует на строку терминала.

Событие:

1. Терминал пишет фразу про комнату.
2. Пауза 0.5–1.0 секунды.
3. Лампа щёлкает.
4. Свет мигает 2–3 раза.

Звук:

- сухой щелчок лампы;
- слабый электрический писк.

Следующий шаг:

```text
Проверить входную дверь.
```

## Beat 4 — Door refuses

Цель игрока:

```text
Проверить входную дверь.
```

Нормальное объяснение:

Смарт-замок завис.

Нарушение:

Текст ошибки выглядит не как обычная ошибка устройства.

Фразы по фазам:

```text
Смарт-замок не отвечает.
```

```text
Ошибка: пользователь находится внутри.
```

```text
Выход не требуется до завершения сборки.
```

Звук:

- короткий electronic reject от двери.

Следующий шаг:

```text
Проверить умную колонку.
```

## Beat 5 — Speaker wakes up

Цель игрока:

```text
Проверить умную колонку.
```

Нормальное объяснение:

Колонка услышала команду случайно.

Нарушение:

Колонка повторяет смысл терминала или отвечает не на вопрос.

Фразы:

```text
Команда не распознана.
```

```text
Распознана команда: продолжать.
```

```text
Режим сна отклонён.
```

Звук:

- speaker wake;
- тихий синтетический голос;
- короткая пауза после фразы.

Следующий шаг:

```text
Вернуться к терминалу.
```

## Beat 6 — Terminal sees the apartment

Цель игрока:

```text
Вернуться к рабочему терминалу.
```

Нормальное объяснение:

Приложение получило данные от умного дома.

Нарушение:

Терминал перечисляет предметы, которые игрок только что проверял.

Пример:

```text
Detected devices:
- desk_lamp: unstable
- smart_lock: waiting
- speaker: listening
```

Звук:

- короткий keyboard burst без видимого ввода;
- мониторный гул на секунду громче.

Следующий шаг:

```text
Отключить рабочий компьютер.
```

## Beat 7 — HUD corruption

Цель игрока:

```text
Отключить рабочий компьютер.
```

Нормальное объяснение:

UI глючит.

Нарушение:

HUD меняет формулировку цели.

Пример:

```text
Отключить рабочий компьютер.
```

становится:

```text
Не отключать рабочий компьютер.
```

или:

```text
Дождаться подтверждения.
```

Звук:

- неправильное уведомление ниже обычного.

Следующий шаг:

```text
Вернуться к терминалу и выбрать действие.
```

## Beat 8 — Final prompt

Цель игрока:

```text
Выбрать: MERGE, REVERT или OFF.
```

Варианты:

| Выбор | Игровой смысл | Эффект |
|---|---|---|
| MERGE | принять изменения | дверь открывается, но за ней слышны уведомления |
| REVERT | откатить изменения | квартира почти нормальна, но в логах остаётся след |
| OFF | выключить питание | свет и звук резко пропадают |

## Очередь реализации

1. Beat 2 + Beat 3: терминал вызывает мигание лампы.
2. Beat 4: дверь получает фазовые сообщения.
3. Beat 5: добавить колонку с одним сообщением.
4. Beat 6: терминал показывает список устройств.
5. Beat 7: HUD corruption.
6. Beat 8: финальный prompt без сложной концовки.
