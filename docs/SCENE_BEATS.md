# SCENE_BEATS.md

## Назначение

Документ раскладывает первый эпизод на короткие игровые сцены. Каждая сцена должна быть достаточно маленькой, чтобы её можно было реализовать отдельным PR.

Каждый beat описывает:

- цель игрока;
- объект взаимодействия;
- нормальное объяснение;
- маленькое нарушение;
- ключевой звук;
- следующий шаг.

## Beat 0 — Late night setup

Цель игрока:

```text
Осмотреть рабочее место.
```

Игрок появляется в квартире ночью рядом с рабочим столом.

Детали:

- монитор светит в комнату;
- на столе есть стикер;
- лампа рядом с монитором;
- входная дверь видна, но пока не главная цель;
- Smart Speaker стоит на полке отдельно от рабочего места;
- HUD показывает первую рабочую задачу.

Звук:

- тихий room tone;
- слабый гул монитора или ПК.

## Beat 1 — Task sticker

Статус: implemented.

Цель игрока:

```text
Прочитать стикер на столе.
```

Нормальное объяснение:

Игрок вспоминает список задач.

Нарушение:

Последняя строка выглядит так, будто её написал не герой.

Пример:

```text
Не доверять ответу, если он слишком уверенный.
```

Следующий шаг:

```text
Открыть рабочий терминал.
```

## Beat 2 — First terminal reply

Статус: implemented.

Цель игрока:

```text
Открыть рабочий терминал.
```

Нормальное объяснение:

Обычный лог сборки и ответ код-агента.

Нарушение:

Предупреждение звучит слишком лично.

Пример:

```text
Build #1047: success
Tests: 0 failed
Warnings: 1

warning: user accepted generated code without reading.
```

Звук:

- procedural `ui_notification_soft` рядом с терминалом;
- позже добавить короткий keyboard tick.

Следующий шаг:

```text
Проверить, почему лампа включилась сама.
```

## Beat 3 — Lamp answers terminal

Статус: implemented.

Цель игрока:

```text
Проверить лампу.
```

Нормальное объяснение:

Глюк умной лампы или электрики.

Нарушение:

Лампа реагирует на строку терминала.

Событие:

1. Терминал показывает личное предупреждение.
2. Срабатывает event `terminal_first_ai_reply`.
3. Играет короткое уведомление рядом с терминалом.
4. Пауза около 0.7 секунды.
5. Лампа щёлкает.
6. Свет мигает 2–3 раза.

Звук:

- procedural `desk_lamp_click` рядом с лампой;
- позже добавить слабый электрический писк.

Следующий шаг:

```text
Проверить входную дверь.
```

## Beat 4 — Door refuses

Статус: planned next.

Цель игрока:

```text
Проверить входную дверь.
```

Нормальное объяснение:

Смарт-замок завис.

Нарушение:

Текст ошибки выглядит не как обычная ошибка устройства.

Фразы по фазам:

```text
Смарт-замок не отвечает.
```

```text
Ошибка: пользователь находится внутри.
```

```text
Выход не требуется до завершения сборки.
```

Звук:

- короткий electronic reject от двери.

Следующий шаг:

```text
Проверить умную колонку.
```

## Beat 5 — Speaker wakes up

Статус: implemented.

Цель игрока:

```text
Проверить умную колонку.
```

Нормальное объяснение:

Колонка услышала команду случайно или не распознала шум.

Нормальное состояние до `terminal_first_ai_reply`:

```text
Команда не распознана.
```

Нарушение после `terminal_first_ai_reply`:

Колонка отвечает так, будто приняла решение за игрока.

Фразы:

```text
Распознана команда: продолжать.
```

```text
Режим сна отклонён.
```

Event:

```text
speaker_wrong_name
```

Реализация:

- `SmartSpeaker.gd` подключается к существующему объекту `SmartSpeaker` через `ApartmentEventController`;
- до `terminal_first_ai_reply` показывает normal body;
- после `terminal_first_ai_reply` показывает strange body;
- повторное взаимодействие показывает repeat body;
- после первого странного ответа цель меняется на возврат к терминалу.

Звук:

- пока нет отдельного `speaker_wake`;
- позже добавить тихий synthetic wake sound.

Следующий шаг:

```text
Вернуться к терминалу.
```

## Beat 6 — Terminal sees the apartment

Цель игрока:

```text
Вернуться к рабочему терминалу.
```

Нормальное объяснение:

Приложение получило данные от умного дома.

Нарушение:

Терминал перечисляет предметы, которые игрок только что проверял.

Пример:

```text
Detected devices:
- desk_lamp: unstable
- smart_lock: waiting
- speaker: listening
```

Звук:

- короткий keyboard burst без видимого ввода;
- мониторный гул на секунду громче.

Следующий шаг:

```text
Отключить рабочий компьютер.
```

## Beat 7 — HUD corruption

Цель игрока:

```text
Отключить рабочий компьютер.
```

Нормальное объяснение:

UI глючит.

Нарушение:

HUD меняет формулировку цели.

Пример:

```text
Отключить рабочий компьютер.
```

становится:

```text
Не отключать рабочий компьютер.
```

или:

```text
Дождаться подтверждения.
```

Звук:

- неправильное уведомление ниже обычного.

Следующий шаг:

```text
Вернуться к терминалу и выбрать действие.
```

## Beat 8 — Final prompt

Цель игрока:

```text
Выбрать: MERGE, REVERT или OFF.
```

Варианты:

| Выбор | Игровой смысл | Эффект |
|---|---|---|
| MERGE | принять изменения | дверь открывается, но за ней слышны уведомления |
| REVERT | откатить изменения | квартира почти нормальна, но в логах остаётся след |
| OFF | выключить питание | свет и звук резко пропадают |

## Очередь реализации

1. Beat 4: дверь получает фазовые сообщения.
2. Beat 6: терминал показывает список устройств.
3. Beat 7: HUD corruption.
4. Beat 8: финальный prompt без сложной концовки.
5. Asset pass: заменить стул, стол/монитор и дверь на совместимые lo-fi / PSX props.
