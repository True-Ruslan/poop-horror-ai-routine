# DECISIONS.md

Журнал архитектурных и продуктовых решений.

## ADR-0001 — Используем Godot 4.7+

Дата: 2026-07-07

Решение:

```text
Шаблон создаётся под Godot 4.7+.
```

Причина:

- актуальная стабильная ветка Godot 4;
- современный 3D renderer;
- GDScript подходит новичкам;
- удобно использовать с ИИ-агентами из-за текстового формата сцен и скриптов.

## ADR-0002 — Базовый шаблон без сторонних зависимостей

Дата: 2026-07-07

Решение:

```text
В стартовый шаблон не включаются COGITO, GodotRetro, Maaack Game Template или другие внешние проекты.
```

Причина:

- проще лицензировать;
- проще понять новичку;
- проще использовать как GitHub Template;
- меньше риска сломать проект;
- внешние проекты можно подключать позже как опциональные интеграции.

## ADR-0003 — GDScript вместо C#

Дата: 2026-07-07

Решение:

```text
Все базовые системы пишутся на GDScript.
```

Причина:

- ниже порог входа;
- лучше совместимость с примерами Godot;
- меньше настройки окружения;
- проще для ИИ-агентов.

## ADR-0004 — Scripted horror events вместо AI-врага

Дата: 2026-07-07

Решение:

```text
В MVP шаблона нет AI-врага. Хоррор строится на scripted events.
```

Причина:

- проще для новичка;
- лучше для коротких psychological horror игр;
- меньше багов;
- игрок всё равно получает напряжение через звук, свет, пространство и события.

## ADR-0005 — Каждый интерактивный объект как отдельная сцена

Дата: 2026-07-07

Решение:

```text
Дверь, записка, выключатель и будущие предметы оформляются как отдельные сцены.
```

Причина:

- легко переиспользовать;
- удобно модифицировать;
- удобно давать задачи ИИ-агенту;
- структура понятна человеку.
