# DECISIONS.md

Журнал архитектурных и продуктовых решений.

## ADR-0001 — Godot 4.7+

Дата: 2026-07-07

Решение: проект разрабатывается на Godot 4.7+ и GDScript.

Причина:

- низкий порог входа;
- текстовый формат сцен удобен для ревью;
- проект проще развивать маленькими итерациями.

## ADR-0002 — Маленькие scripted events

Дата: 2026-07-07

Решение: хоррор в MVP строится на маленьких scripted events, а не на сложных системах поведения.

Причина:

- меньше багов;
- проще тестировать;
- лучше подходит для короткого хоррора.

## ADR-0003 — Интерактивные объекты как отдельные сцены

Дата: 2026-07-07

Решение: дверь, записка, выключатель, монитор и будущие предметы оформляются как отдельные сцены.

Причина:

- легко переиспользовать;
- удобно модифицировать;
- удобно давать маленькие задачи ИИ-агенту.

## ADR-0004 — Проект становится конкретной игрой

Дата: 2026-07-07

Решение: репозиторий развивается как `AI Routine: Last Commit`, а не только как общий шаблон.

Причина:

- пользователь хочет собственный мини-хоррор;
- конкретный сценарий помогает двигаться быстрее;
- первая квартира разработчика даёт понятный playable slice.

## ADR-0005 — Первая локация: квартира разработчика

Дата: 2026-07-07

Решение: `Main.tscn` запускает `DeveloperApartment.tscn`.

Причина:

- локация соответствует теме рабочей рутины;
- она маленькая и понятная;
- в ней достаточно объектов для первой версии.

## ADR-0006 — Сначала собственная геометрия и простые лицензии

Дата: 2026-07-07

Решение: на старте используем примитивы Godot и позже добавляем только ассеты с понятными лицензиями.

Причина:

- проект планируется для публикации;
- проще контролировать права;
- проще вести `docs/CREDITS.md`.

## ADR-0007 — Периодически сверяемся с upstream-шаблоном

Дата: 2026-07-07

Решение: проект должен периодически проверять изменения в `True-Ruslan/godot-simple-tamplate`, но не переносить их автоматически.

Причина:

- исходный шаблон развивается отдельно;
- новые checklist/prompt/docs могут быть полезны;
- текущая игра уже отличается от generic-шаблона;
- перенос изменений должен быть осознанным и безопасным.
