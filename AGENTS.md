# AGENTS.md

Инструкции для Codex, Cursor, Claude Code и других ИИ-агентов.

## Роль агента

Ты работаешь с Godot-проектом короткой атмосферной хоррор-игры. Твоя задача — помогать развивать шаблон итерационно, безопасно и понятно.

## Обязательные источники контекста

Перед изменениями прочитай:

```text
docs/CONTEXT.md
docs/TASKS.md
docs/DECISIONS.md
docs/CREDITS.md
```

Если задача связана со сценарием, также прочитай:

```text
docs/SCENARIO_TEMPLATE.md
```

## Главные правила

1. Не генерируй всю игру целиком.
2. Не ломай существующий запуск проекта.
3. После каждой задачи проект должен запускаться.
4. Не добавляй внешние зависимости без записи в `docs/CREDITS.md` и `docs/DECISIONS.md`.
5. Не добавляй GPL/AGPL/LGPL-код или ассеты без явного решения человека.
6. Не добавляй тяжёлые системы раньше, чем они реально нужны.
7. Не добавляй боевую систему, оружие или AI-врага, если задача этого прямо не требует.
8. Любая новая механика должна иметь минимальный тестовый пример в демо-сцене или отдельной test-сцене.
9. Все новые файлы должны находиться в существующей структуре `game/` или `docs/`.
10. Если меняешь архитектуру, добавь запись в `docs/DECISIONS.md`.
11. Если меняешь поведение проекта, добавь запись в `docs/CHANGELOG.md`.
12. Если выполняешь задачу, обнови `docs/TASKS.md`.

## Стиль кода

- Язык: GDScript.
- Godot: 4.7+.
- Имена файлов: `PascalCase.gd` для классов/объектов, `snake_case` допустим только для данных.
- Узлы сцен должны иметь понятные имена: `Player`, `InteractionRaycast`, `ObjectiveManager`, `Door_MainHall`.
- Не использовать магические строки без необходимости.
- Сигналы должны называться глаголами или событиями: `objective_changed`, `message_requested`, `interaction_prompt_changed`.
- Экспортируемые параметры помечать `@export`.
- Сложную логику делить на небольшие компоненты.

## Definition of Done для любой задачи

Задача считается завершённой, если:

- проект открывается в Godot;
- основная сцена запускается;
- нет ошибок в Output при старте;
- добавленная механика проверяется вручную за 1–2 минуты;
- документация обновлена, если изменилось поведение;
- изменения минимальны и относятся к задаче.

## Запрещённые действия

Не делай это без отдельного разрешения:

- миграция проекта на C#;
- добавление сторонних плагинов;
- добавление сетевого режима;
- добавление Steam SDK;
- добавление AI-врага;
- генерация больших 3D-локаций одним коммитом;
- переписывание всех существующих скриптов;
- изменение структуры проекта без записи в `docs/DECISIONS.md`.
